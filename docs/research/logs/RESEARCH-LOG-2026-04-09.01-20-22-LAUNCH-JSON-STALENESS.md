# Research Log: launch.json Port Staleness

**Story:** 2026-04-09.01-20-22 — Fix generate-launch-json.sh port staleness
**Type:** SDLC

## Findings

### Root Cause
`launch.json` is generated by `scripts/generate-launch-json.sh` and consumed by Claude Preview MCP (`preview_start`). Both call sites (`prompt-init-gate.sh`, `/init` skill) guarded generation with existence checks — only creating the file if missing. Once created, it was never regenerated even when `.env` changed.

### Evidence
- `.env` had `PORT=9001` but `launch.json` had port `9003` (stale from prior agent port assignment)
- Frontend started with wrong `VITE_API_BASE_URL`, hitting hub API instead of CTEDemo backend
- Additional: file was tracked in git, propagating stale values across branches

### Fix Applied
1. Removed existence guards — always regenerate from `.env`
2. Added `.claude/launch.json` to `.gitignore`
3. Untracked from git via `git rm --cached`

### Consumer Analysis
- Only consumer: Claude Preview MCP (`preview_start` tool)
- No shell scripts, hooks, or skills read `launch.json` programmatically
- External `agents.sh` (not in repo) calls the script directly, unaffected by this change
